# Submit-testing

这是一个用于展示送检测试记录的静态页面。

## 使用方式

- 把每条测试记录写成 `.md` 文件，放到 [`testing/`](./testing) 目录下。
- 部署到 GitHub Pages 后，首页 [`index.html`](./index.html) 会自动读取 `testing/` 目录中的所有 Markdown 文件并展示。
- 页面会按文件名排序显示记录列表，点击某条记录即可查看正文。

## 记录格式建议

你可以按自己习惯写 Markdown，常见字段例如：

```md
# 2026-06-30

- 项目名
- 测试人
- 结果
- 备注
```

只要文件后缀是 `.md`，页面就会自动识别。
